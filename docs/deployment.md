# Deployment

Deployment uses GitHub Actions OIDC and Bicep.

## Required GitHub Environment Secrets

- `AZURE_CLIENT_ID`
- `AZURE_TENANT_ID`
- `AZURE_SUBSCRIPTION_ID`
- `SQL_ADMIN_PASSWORD`

The federated credential must allow the repository environment to log in to Azure without a client secret.

## Infrastructure

`infra/main.bicep` provisions:

- Linux App Service plan
- App Service with system-assigned managed identity
- Staging deployment slot
- Key Vault with RBAC
- Log Analytics
- Application Insights
- Azure SQL server and database
- VNet integration subnet
- SQL private endpoint subnet
- SQL private DNS zone and VNet link

Use environment parameter files from `infra/environments`.

```powershell
$env:SQL_ADMIN_PASSWORD = "<secret>"
az deployment group what-if --resource-group <rg> --template-file infra/main.bicep --parameters infra/environments/dev.bicepparam
az deployment group create --resource-group <rg> --template-file infra/main.bicep --parameters infra/environments/dev.bicepparam
```

## Application

`deploy.yml` publishes the API, deploys to the staging slot, smoke tests `/health/live`, and swaps the slot into production.

## Database

Migration scripts are generated by `migration-script.yml`. Because Azure SQL is private, apply the reviewed SQL from a self-hosted runner inside the VNet, an approved operator workstation, or a DBA-controlled path.
